# Лабораторная работа №7. Управление зависимостями с помощью Hunter и автоматизация релизов в GitHub Actions

* Репозиторий: [lab07](https://github.com/sofarun313131/lab07)

---

## Цель работы
Изучить систему управления зависимостями Hunter для интеграции сторонних библиотек (Google Test) в C++ проект. Настроить автоматическую сборку, тестирование и генерацию инсталляционных пакетов с помощью CPack и GitHub Actions.

---

## Выполнение работы
1. В файле CMakeLists.txt был подключен менеджер пакетов Hunter и зарегистрирована зависимость Google Test.
2. Локальная сборка проекта выполнена с флагом CXXFLAGS="${CXXFLAGS} -Wno-error=maybe-uninitialized" для обхода предупреждений в коде старой версии GTest.
3. Настроен сценарий автоматической сборки .github/workflows/actions.yml. Флаги компиляции были прописаны в блоке env на сервере.
4. В удалённый репозиторий отправлен релизный тег v0.7.0. Сборка на GitHub Actions завершилась успешно (зеленый маркер), артефакты .deb, .rpm, .tar.gz прикрепились к релизу.

---

## Вывод
В ходе лабораторной работы были освоены принципы управления зависимостями в C++ с помощью Hunter и настроен автоматический CI/CD конвейер в GitHub Actions.
